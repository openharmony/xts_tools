## 一、行为差异与规范一致性概览

### 观察点 N：接口方法声明语法 ⭐⭐⭐ HIGH

**用例：** ITF_10_05_001~020 系列

**实际行为：**
```typescript
// ArkTS 接口方法声明
interface I {
  method(): void              // 抽象方法（默认）
  defaultMethod(): void {    // 默认方法（有实现）
    /* 默认实现 */
  }
}

// 类实现接口
class C implements I {
  method(): void { /* 必须实现 */ }
  // defaultMethod 可选实现（继承默认实现）
}
```

**对比：**
| 维度 | ArkTS | Java | Swift | TypeScript |
|------|-------|------|-------|------------|
| 抽象方法 | ✅ 接口方法默认抽象 | ✅ 接口方法默认抽象 | ✅ protocol 方法声明 | ✅ 接口方法声明 |
| 带参数方法 | ✅ | ✅ | ✅ | ✅ |
| 带返回值方法 | ✅ | ✅ | ✅ | ✅ |
| void 方法 | ✅ | ✅ | ✅ | ✅ |
| 类实现接口 | ✅ `implements` 关键字 | ✅ `implements` | ✅ `: Protocol` | ✅ `implements` |
| 多接口实现 | ✅ 支持 | ✅ 支持 | ✅ 支持 | ✅ 支持 |
| 缺少实现 | ❌ 编译拒绝 | ❌ 编译拒绝 | ❌ 编译拒绝 | ❌ 编译拒绝 |
| 签名不匹配 | ❌ 编译拒绝 | ❌ 编译拒绝 | ❌ 编译拒绝 | ❌ 编译拒绝 |

**影响：** ArkTS 的接口方法语法与 Java/TypeScript 高度一致。

**评价：** 无问题，设计合理。
