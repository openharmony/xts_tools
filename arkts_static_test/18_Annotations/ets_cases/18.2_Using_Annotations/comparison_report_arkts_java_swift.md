# 18.2 Using Annotations - 跨语言对比报告

## 1. 概览

| 语言 | 定位 |
|------|------|
| ArkTS | 使用 `@Anno` / `@Anno({...})` 语法，支持目标包括类、方法、字段、参数、lambda、变量等 |
| Java | 使用 `@Anno` / `@Anno(...)` 语法，支持类、方法、字段、参数，不支持 lambda 和变量 |
| Swift | 无原生注解，使用属性包装器和宏 |

## 4. 三环境实测结果

所有 23 个 ArkTS 用例均实际通过 `es2panda + ark` 验证。Java 等价代码通过 `javac + java` 验证。

## 5. 综合评分

| 维度 | 评分 | 说明 |
|------|------|------|
| 语法一致性 | ★★★★★ | 与 Java 高度一致 |
| 应用目标广度 | ★★★★★ | 支持 lambda 和变量，比 Java 更广 |
| 错误诊断 | ★★★★★ | ESE0041/ESE0044/ESY0028 等清晰 |
| 常量约束 | ★★★★★ | 严格检查常量表达式 |

## 6. 核心结论

1. **ArkTS 注解使用语法与 Java 几乎一致**：支持 `@Anno`、`@Anno()`、`@Anno({...})` 三种形式。
2. **ArkTS 应用目标比 Java 更广**：支持 lambda 和变量声明（需 SOURCE 策略）。
3. **所有规则实现与 spec 一致**：重复注解禁止、必填字段检查、常量表达式要求等全部正确。
4. **错误诊断良好**：每个违规场景都有明确的错误码和消息。
