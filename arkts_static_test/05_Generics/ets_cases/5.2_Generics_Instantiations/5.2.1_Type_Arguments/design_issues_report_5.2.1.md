## 二、语言设计差异

### 差异 F：丰富的类型参数支持 ⭐⭐⭐ HIGH

**说明：** ArkTS 支持数字、联合、数组、元组、函数等多种类型作为类型参数，比 Java 更丰富，与 TypeScript 高度一致。此行为符合 ArkTS Spec，非缺陷。

**对比：**
| 维度 | ArkTS | Java | Swift | TypeScript |
|------|-------|------|-------|------------|
| 数字类型参数 | ✅ `Box<int>` | ✅ `Box<Integer>` | ✅ `Box<Int>` | ✅ `Box<number>` |
| 联合类型参数 | ✅ `Box<A \| B>` | ❌ 不支持联合 | ❌ 不支持联合 | ✅ `Box<A \| B>` |
| 数组类型参数 | ✅ `Box<int[]>` | ✅ `Box<int[]>` | ✅ `Box<[Int]>` | ✅ `Box<number[]>` |
| 元组类型参数 | ✅ `Box<[int, string]>` | ❌ 不支持 | ✅ `Box<(Int, String)>` | ✅ `Box<[number, string]>` |
| 函数类型参数 | ✅ `Box<() => void>` | ❌ 不支持（函数类型非一等） | ✅ `Box<() -> Void>` | ✅ `Box<() => void>` |
| 无效类型参数 | ❌ 编译拒绝 | ❌ 编译拒绝 | ❌ 编译拒绝 | ❌ 编译拒绝 |
