## 二、语言设计差异

### 差异 D：声明位变体标注 `in`/`out` 与 Java use-site 差异 ⭐⭐⭐ HIGH

**说明：** ArkTS 使用声明位变体标注（`in`/`out`），Java 使用 use-site 通配符（`? extends`/`? super`）。这是语言设计层面的选择差异，非缺陷。ArkTS 的声明位变体更简洁，Java 的 use-site 更灵活但使用更繁琐。

**对比：**
| 维度 | ArkTS | Java | Swift | TypeScript |
|------|-------|------|-------|------------|
| 变体机制 | 声明位 `in`/`out` | use-site `? extends`/`? super` | 集合协变 | 结构类型系统 |
| 协变返回值 | ✅ `out T` 允许 | ✅ `? extends T` | ✅ | ✅ |
| 逆变参数 | ✅ `in T` 允许 | ✅ `? super T` | ✅ | ✅ |
| 不变 | ✅ 默认 | ✅ 默认 | ✅ `var` 参数 | ✅ |
| 只读字段协变 | ✅ `out T` + readonly | ✅ `? extends` | ✅ `let` | ✅ |
| 构造函数中变体 | ✅ out 允许 | N/A | ✅ | N/A |
