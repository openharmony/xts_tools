## 二、语言设计差异

### 差异 A：keyof 约束错误信息仅显示第一个 key ⭐⭐ MEDIUM

**用例：** GEN_05_01_023_FAIL_KEYOF_CONSTRAINT

**实际行为：**
```typescript
// 约束为 extends keyof A（其中 A 有 f1/f2/f3 三个字段）
// 错误信息显示 '"f0"' should be a subtype of '"f1"' constraint
// 只列出了第一个 key "f1" 而非完整的 "f1"|"f2"|"f3"
```

**说明：** 此问题属于错误信息质量优化，非 Spec 语义不一致。Spec 未规定错误信息的详细格式。

**对比：**
| 语言 | keyof 等效机制 | 错误信息 |
|------|---------------|---------|
| ArkTS | `extends keyof T` | 仅显示第一个 key（待优化） |
| Java | 不支持 keyof 约束 | N/A |
| Swift | 无等效 keyof | N/A |
| TypeScript | `extends keyof T` | 显示完整联合类型 |

**建议：** 优化错误信息格式，显示完整的 keyof 联合类型。
