## 一、与业界静态语言的差异点

### 设计观察：字段方法同名冲突 — ArkTS独有约束

**用例：** CLS_09_03_007_FAIL_FIELD_METHOD_NAME_CONFLICT
**对比：** ArkTS 规定同一 class 声明中字段名与方法名互斥。Java 的字段和方法分属不同命名空间（不冲突），Swift 通过类型系统区分（允许但有歧义）。

**影响：** 此约束减少了意外遮蔽风险，但也降低了灵活性（如 getter-like 方法命名）。这是 spec 明确的设计选择，非缺陷。

## 四、编译规则正确性评估

| 规则 | spec 描述 | 实测 | 结论 |
|------|----------|------|------|
| implements 仅接受接口 | implements 目标必须是接口名 | 编译拒绝不可访问接口 | ✅ 一致 |
| 同泛型不同实例化 | 同泛型接口不同实例化冲突 | 编译拒绝 (CLS_09_03_006) | ✅ 一致 |
| 字段方法同名 | 同声明中字段方法同名冲突 | 编译拒绝 (CLS_09_03_007) | ✅ 一致 |
| 未实现接口方法 | 类必须实现所有接口方法 | 编译拒绝 (CLS_09_03_008) | ✅ 一致 |
| 重复接口 | 重复接口名被忽略 | 编译通过 (CLS_09_03_004) | ✅ 一致 |
| 接口方法派发 | 通过接口引用调用方法 | 运行时通过 (CLS_09_03_009) | ✅ 一致 |
