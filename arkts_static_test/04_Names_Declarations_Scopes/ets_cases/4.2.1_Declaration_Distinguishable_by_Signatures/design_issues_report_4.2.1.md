## 一、行为差异与规范一致性概览

### 差异 A-4.2.1：声明可区分性检查严格（语言设计差异）

**用例：** NAM_04_02_01_100_FAIL_indistinguishable_signature

**实际行为：**
```typescript
function foo(x: int): void {}
function foo(x: int): void {}  // ❌ 编译错误：不可区分的签名
```

**对比：**
| 语言 | 声明可区分性检查 |
|------|-----------------|
| ArkTS | 严格（基于签名） |
| Java | 严格（基于方法签名） |
| Swift | 严格（基于参数标签） |
| TypeScript | 宽松（声明合并） |

**影响：** 无负面影响，防止重复声明。

**评价：** 设计合理，与静态类型语言一致。

| 维度 | ArkTS | Java | Swift | TypeScript |

| 同一作用域同名 | ❌ 禁止（除非可区分） | ❌ 禁止（除非重载） | ❌ 禁止（除非参数标签不同） | ✅ 允许（声明合并） |
| 可区分性基于 | 参数类型序列 | 参数类型/数量 | 参数标签 | N/A（合并） |
| 返回类型影响 | ❌ 不影响 | ❌ 不影响 | ❌ 不影响 | N/A |

### 2.2 重载规则

| 维度 | ArkTS | Java | Swift | TypeScript |
|------|-------|------|-------|------------|

| 泛型重载 | ✅ 支持 | ✅ 支持 | ✅ 支持 | N/A |
