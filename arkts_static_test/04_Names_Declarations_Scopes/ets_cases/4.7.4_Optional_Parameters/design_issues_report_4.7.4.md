## 一、行为差异与规范一致性概览

### 差异 A-4.7.4：可选参数必须位于必需参数之后（语言设计差异）

**Observation:** ArkTS 要求可选参数必须位于必需参数之后，而 TypeScript 允许任意顺序。

**对比：**
| 语言 | 可选参数位置 |
|------|-------------|
| ArkTS | 必须位于必需参数之后 |
| Java | N/A（不支持可选参数） |
| Swift | 可位于任意位置（有默认值） |
| TypeScript | 可位于任意位置 |

**影响：** 无重大影响，符合函数调用惯例。

**评价：** 设计合理，非缺陷。
