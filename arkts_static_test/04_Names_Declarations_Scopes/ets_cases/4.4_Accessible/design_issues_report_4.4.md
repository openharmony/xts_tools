## 一、行为差异与规范一致性概览

### 差异 A-4.4：块级作用域访问控制（语言设计差异）

**Observation:** ArkTS 的块级作用域（let/const）自动限制变量在块内访问，而 Java 的块级作用域更灵活（可以在块内声明，块外不可访问）。

**对比：**
| 语言 | 块级作用域访问控制 |
|------|-------------------|
| ArkTS | let/const 自动块级作用域 |
| Java | 块内声明，块外不可访问 |
| Swift | let/var 块级作用域 |
| TypeScript | let/const 块级作用域 |

**影响：** 无重大影响，符合现代语言设计。

**评价：** 符合 ArkTS 简化设计哲学，非缺陷。

## 二、跨语言对比详细分析

### 2.1 作用域可访问性

| 维度 | ArkTS | Java | Swift | TypeScript |
|------|-------|------|-------|------------|
| 模块级可访问性 | 文件级可见 | 包级可见 | 文件级可见（默认 internal） | 文件级可见（模块化） |
| 类成员可访问性 | public/private/protected | public/private/protected/package-private | public/internal/private/fileprivate | public/private/protected（ES6） |
| 块级可访问性 | let/const 块级 | 块内声明不可外部访问 | let/var 块级 | let/const 块级 |

### 2.2 跨作用域访问

| 维度 | ArkTS | Java | Swift | TypeScript |
|------|-------|------|-------|------------|
| 嵌套函数访问 | ❌ 不支持嵌套函数 | ✅ 支持（JDK 16+） | ✅ 支持（捕获语义） | ✅ 支持 |
| 闭包捕获 | ✅ 支持 | ✅ 支持（effectively final） | ✅ 支持（值捕获/引用捕获） | ✅ 支持 |
