## 一、行为差异与规范一致性概览

### 差异 A-4.7：不支持嵌套函数（语言设计差异）

**Observation:** ArkTS 不支持在函数内部声明函数，而 Java（JDK 16+）、Swift、TypeScript 都支持嵌套函数。

**对比：**
| 语言 | 嵌套函数支持 |
|------|-------------|
| ArkTS | ❌ 编译拒绝 |
| Java | ✅ 支持（JDK 16+） |
| Swift | ✅ 支持（捕获语义） |
| TypeScript | ✅ 支持 |

**影响：** 无法组织复杂逻辑，所有函数必须放在顶层，限制了代码组织能力。

**评价：** 符合 ArkTS 静态化、简化设计哲学，但限制了表达能力。

**建议：** 评估是否允许嵌套函数（提升代码组织能力）。

## 二、跨语言对比详细分析

### 2.1 函数声明规则

| 维度 | ArkTS | Java | Swift | TypeScript |
|------|-------|------|-------|------------|
| 顶层函数 | ✅ 支持 | ❌ 必须位于类内 | ✅ 支持 | ✅ 支持 |
| 嵌套函数 | ❌ 禁止 | ✅ 支持（JDK 16+） | ✅ 支持 | ✅ 支持 |
| 函数重载 | ✅ 支持（可区分签名） | ✅ 支持 | ✅ 支持（参数标签区分） | ❌ 不支持（声明合并） |
| native 函数 | ✅ 支持 | ✅ 支持 | ❌ 无直接等效 | ❌ 无 |

### 2.2 函数声明语法

| 维度 | ArkTS | Java | Swift | TypeScript |
|------|-------|------|-------|------------|
| 函数声明关键字 | `function` | 返回类型 + 方法名 | `func` | `function` 或箭头 |
| 箭头函数 | ✅ 支持 | ⚠️ Lambda（SAM） | ✅ 支持（闭包） | ✅ 支持 |
| 函数类型 | ✅ 支持 | ⚠️ 函数式接口 | ✅ 支持 | ✅ 支持 |
