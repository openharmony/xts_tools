## 一、行为差异与规范一致性概览

### 差异 A-4.6.2：const 必须初始化（语言设计差异）

**用例：** NAM_04_06_02_100_FAIL_const_no_init

**实际行为：**
```typescript
const PI: number;  // ❌ 编译错误：const 必须初始化
```

**对比：**
| 语言 | const/final 初始化要求 |
|------|----------------------|
| ArkTS | ✅ 声明时必须初始化 |
| Java | ✅ 构造器中必须初始化 |
| Swift | ✅ 声明时必须初始化 |
| TypeScript | ✅ 声明时必须初始化 |

**影响：** 无负面影响，符合常量语义。

**评价：** 设计合理，与主流语言一致。

## 二、跨语言对比详细分析

### 2.1 常量声明规则

| 维度 | ArkTS | Java | Swift | TypeScript |
|------|-------|------|-------|------------|
| 声明关键字 | `const` | `final` | `let`（值不可变） | `const` |
| 初始化时机 | 声明时 | 声明时或构造器 | 声明时 | 声明时 |
| 编译期常量 | ✅ 要求 | ⚠️ 不一定 | ✅ 要求 | ✅ 要求 |
| 运行时常量 | ❌ 不支持 | ✅ `final` 变量 | ❌ 不支持 | ❌ 不支持 |

### 2.2 常量类型推断

| 维度 | ArkTS | Java | Swift | TypeScript |
|------|-------|------|-------|------------|
| 类型推断 | ✅ 支持 | ✅ 支持（`var`） | ✅ 支持 | ✅ 支持 |
| 字面量类型保留 | ✅ 保留 | N/A | ✅ 保留 | ✅ 保留 |
| 对象字面量 | ⚠️ 部分支持 | N/A | ✅ 支持 | ✅ 支持 |
