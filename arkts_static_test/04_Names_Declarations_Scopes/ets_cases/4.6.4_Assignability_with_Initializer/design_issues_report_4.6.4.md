## 一、行为差异与规范一致性概览

### 差异 A-4.6.4：可赋值性规则严格（语言设计差异）

**Observation:** ArkTS 的可赋值性规则较为严格，而 TypeScript 允许更多隐式类型转换。

**对比：**
| 语言 | 可赋值性规则 |
|------|-------------|
| ArkTS | 严格（需精确类型匹配或显式转换） |
| Java | 严格（自动拓宽转换） |
| Swift | 严格（可选类型需显式处理） |
| TypeScript | 宽松（结构类型兼容） |

**影响：** 需要更多显式类型转换，但提升了类型安全性。

**评价：** 符合 ArkTS 静态化、安全设计哲学，非缺陷。
