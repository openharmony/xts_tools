## 一、行为差异与规范一致性概览

### 说明 D-4.6：`int`/`double` 推断 — 规范一致行为

**根据 ArkTS 规范第 4 章（Names §4.6.5）及 types.md 定义，`int`、`double`、`byte`、`short`、`long`、`float`、`char`、`boolean` 等均为预设基本类型的正式名称，字面量推断规则（let 推断为 int、三元推断为 int|double）已在 §4.6.5 Type Inference from Initializer 中通过示例给出。因此该行为符合规范，不属于设计缺陷或 Spec/实现不一致。**

## 五、4.6 章节核心结论

| 维度 | 评价 |
|------|------|
| 变量声明 | ⭐⭐⭐⭐ 完整支持，与 TypeScript 一致 |
| 常量声明 | ⭐⭐⭐⭐ 编译期常量要求严格 |
| 类型推断 | ⭐⭐⭐⭐ 支持良好，但 `int` 类型未定义 |
| 与 TypeScript 兼容 | ⭐⭐⭐⭐ `let`/`const` 语法完全一致 |
| Spec 完整性 | ⭐⭐ `int`/`double` 等基础类型未定义 |

## 六、建议改进

### 短期
1. Spec 明确定义 `int`、`double`、`char`、`byte` 等基础类型

### 中期
2. 统一类型推断策略（明确何时推断为 `int` vs `number`）

### 长期
3. 考虑支持运行时常量（如 Java 的 `final` 变量）
