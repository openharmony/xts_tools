## 一、行为差异与规范一致性概览

### 差异 A-4.7.1：函数签名可区分性检查严格（语言设计差异）

**Observation:** ArkTS 要求重载函数的签名必须可区分（基于参数类型序列），而 TypeScript 不支持重载（使用声明合并）。

**对比：**
| 语言 | 函数签名可区分性 |
|------|-----------------|
| ArkTS | 基于参数类型序列 |
| Java | 基于参数类型/数量 |
| Swift | 基于参数类型/标签 |
| TypeScript | N/A（声明合并） |

**影响：** 无负面影响，严格的检查防止运行时歧义。

**评价：** 设计良好，符合静态类型语言设计哲学。
