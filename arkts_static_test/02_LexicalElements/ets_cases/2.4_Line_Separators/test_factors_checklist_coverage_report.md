由于 Java 和 Swift 不直接支持 LS (U+2028) 和 PS (U+2029) 作为源代码换行符，存在以下限制：

| 限制场景 | ArkTS (WSL) | Java (模拟) | Swift (模拟) | 实际测试状态 |
|---------|-------------|-------------|-------------|-------------|
| 文件原生 LS 字面量 | ✅ | ❌ | ❌ | ⚠️ 仅ArkTS测试 |
| 文件原生 PS 字面量 | ✅ | ❌ | ❌ | ⚠️ 仅ArkTS测试 |
| LS/PS 序列交织 | ✅ | ❌ | ❌ | ⚠️ 仅ArkTS测试 |
| LF/CR 序列等价 | ✅ | ✅ | ✅ | ✅ 三语言一致 |
| 运行时语义 | ✅ | ✅ | ✅ | ✅ 三语言一致 |

### Java 测试策略

1. **不支持 LS/PS 场景**
   - 使用 `\n` 和 `\r\n` 替代 LS 和 PS 测试
   - 验证 LF/CR 序列的运行时行为

2. **验证内容**
   - 024：LF-only 风格运算 ✅
   - 025：CRLF 风格运算 ✅
   - 026：多行注释跨行 ✅
   - 027：模板字符串换行内容 ✅
   - 028：连续空行 ✅
   - 029：行终止符序列等价 ✅

### Swift 测试策略

与 Java 策略一致，但不使用 `System.lineSeparator()`，保持跨语言可移植性。
