# 2.4 Line Separators - 测试设计思维导图（v1.1 - 标准化 + 补充cross_lang_verify目录）

## 概述
Line separators 是词法输入元素，既分隔 Token，也将 Unicode 输入字符序列划分为行。包含 4 种：
- Newline character (U+000A, LF)
- Carriage return character (U+000D, CR)
- Line separator character (U+2028, LS)
- Paragraph separator character (U+2029, PS)

**关键规则：**
- Line separators 提升源码可读性
- **任意连续行终止符序列视为单个分隔符**
- 通常被当作 white space 处理，**除非在有特殊含义的场景**（详见 §2.11 Semicolons）

> ⚠️ 重要说明：本测试设计参考 ArkTS Static Language Specification §2.4，并结合 TESTING_PROCESS_GUIDE.md 要求，标准化为所有章节的统一格式。所有测试用例均已通过实际 ark VM 运行验证。

## 分类说明
- **compile-pass**：合法行终止符使用必须编译通过
- **compile-fail**：行终止符在 Token 内部未转义必须编译失败
- **runtime**：行终止符变化不影响运行时语义，必须实际执行验证

## 跨语言差异说明

### Java/Swift 限制

| 限制项 | 说明 | 影响 |
|--------|------|------|
| 仅支持 LF/CR | LS (U+2028) 和 PS (U+2029) 语法错误 | 需使用 `\u2028` 转义 |
| 特殊字符使用 | 零宽字符需要转义形式 | 测试代码差异 |
| 换行符 API | `System.lineSeparator()` | 可移植性差异 |

### ArkTS 优势

| 优势项 | 说明 |
|--------|------|
| UTF-8 文件原生支持 | 可直接使用任意 Unicode 行终止符 |
| 兼容性（JS 源） | 沿袭 ECMAScript 允许 LS/PS 的设计 |
| 统一处理 | 简化文件格式要求 |
