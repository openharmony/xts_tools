# 2.9.1 Numeric Literals - 测试设计思维导图（v1.1 - 补充cross_lang_verify目录）

## 概述
Numeric literals 包括整数和浮点字面量。ArkTS 支持多种进制、下划线分隔符、类型后缀等。

## 全覆盖测试矩阵

### 维度 1：整数字面量

#### A. 进制
- **A1** 十进制：42, 0, -1, 2147483647 (001)
- **A2** 十六进制：0xFF, 0x00, 0xDEADBEEF (002)
- **A3** 八进制：0o77, 0o0, 0o377 (003)
- **A4** 二进制：0b1010, 0b0, 0b11111111 (004)

#### B. 下划线分隔符
- **B1** 十进制带下划线：1_000_000, 1_2_3 (005)
- **B2** 十六进制带下划线：0xFF_FF (006)

#### C. 类型后缀
- **C1** float 后缀 f：3.14f (007)
- **C2** bigint 后缀 n：123n (008)

### 维度 2：浮点字面量

#### D. 格式
- **D1** 标准浮点：3.14, 0.5, .5 (009)
- **D2** 科学计数法：1e10, 1.5e-5, 2E3 (010)
- **D3** 无前导零浮点：.5, .25 (011)

### 维度 3：类型推断

#### E. 字面量类型推断
- **E1** 整数推断 int：let x = 42 (012)
- **E2** 浮点推断 double：let x = 3.14 (013)
- **E3** 大整数推断 long：let x = 2147483648 (014)
- **E4** bigint 字面量：let x = 123n (015)

### 维度 4：边界值

#### F. 数值边界
- **F1** int 最大值 2147483647 (016)
- **F2** int 最小值 -2147483648 (017)
- **F3** long 超出 int 范围 (018)

### 维度 5：compile-fail
- 019: 前导零 0123（八进制旧语法不支持）
- 020: 0x 后无数字
- 021: 0b 后非二进制
- 022: 0o 后非八进制

### 维度 6：runtime
- 023: 各进制值相等
- 024: 浮点运算
- 025: bigint 运算
- 026: 下划线不影响值
- 027: 科学计数法值

## 编号规划
- compile-pass: 001~018 (18个)
- compile-fail: 019~022 (4个)
- runtime: 023~027 (5个)
