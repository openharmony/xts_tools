## 六、语言设计差异详情

### 差异 A：比较运算符差异

**现象：**
- ArkTS: `===` / `!==`
- Java: `==` / `!=`
- Swift: `==` / `!=`

**影响：**
- 语法层面差异，运行时语义一致
- 不影响布尔值的相等性判断结果

**规范依据：**
- ArkTS spec §2.8.1: 严格相等运算符用于布尔比较
- Java JLS §15.21.3: 相等运算符
- Swift Language Reference: 相等运算符

**分类：** 符合 ArkTS spec 的语言设计差异，非缺陷

---

### 差异 B：类型名称差异

**现象：**
- ArkTS: `boolean`
- Java: `boolean`
- Swift: `Bool`

**影响：**
- 类型系统命名约定不同
- 不影响运行时行为

**规范依据：**
- ArkTS spec §2.9.5: 布尔类型定义
- Java JLS §4.2.5: boolean 类型
- Swift Language Reference: Bool 类型

**分类：** 符合 ArkTS spec 的语言设计差异，非缺陷

---

### 差异 C：双重 NOT 语法差异

**现象：**
- ArkTS: `!!true` (合法)
- Java: `!!true` (合法)
- Swift: `!(!true)` (必须使用括号)

**影响：**
- 语法层面差异
- 运算结果一致

**规范依据：**
- ArkTS spec §2.8.2: 前缀运算符
- Java JLS §15.15.2: 前缀递减运算符
- Swift Language Reference: 逻辑运算符

**分类：** 符合 ArkTS spec 的语言设计差异，非缺陷
