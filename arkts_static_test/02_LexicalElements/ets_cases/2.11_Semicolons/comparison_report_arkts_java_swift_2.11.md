# 2.11 Semicolons - ArkTS/Java/Swift 对比报告

## 概述
本报告对比 ArkTS、Java、Swift 三种语言在 semicolons 测试中的行为差异。

---

## 语言差异对比

| 特性 | ArkTS | Java | Swift |
|------|-------|------|-------|
| 行分隔符终止 | ✅ 支持 | ❌ 必须分号 | ✅ 支持 |
| 分号分隔 | ✅ 支持 | ✅ 支持 | ✅ 支持 |
| 歧义避免 | ✅ 支持 | ✅ 支持 | ✅ 支持 |
| 缺少分号 | ✅ 编译失败 | ✅ 编译失败 | ✅ 编译失败 |

---

## 测试用例对比

### 001: 变量声明由行分隔符终止
| 语言 | 代码 | 结果 |
|------|------|------|
| ArkTS | `let x: int = 1` | ✅ |
| Java | `int x = 1;` | ✅ |
| Swift | `let x = 1` | ✅ |

### 004: 单行多个变量声明
| 语言 | 代码 | 结果 |
|------|------|------|
| ArkTS | `let x: int = 1; let y: int = 2` | ✅ |
| Java | `int x = 1; int y = 2;` | ✅ |
| Swift | `let x = 1; let y = 2` | ✅ |

### 007: 分号避免表达式歧义
| 语言 | 代码 | 结果 |
|------|------|------|
| ArkTS | `i; -i++` | ✅ |
| Java | `i; -i++;` | ✅ |
| Swift | `i; -i += 1` | ✅ |

### 009: 缺少分号导致编译失败
| 语言 | 代码 | 结果 |
|------|------|------|
| ArkTS | `let x: int = 1 let y: int = 2` | ✅ 编译失败 |
| Java | `int x = 1 int y = 2;` | ✅ 编译失败 |
| Swift | `let x = 1 let y = 2` | ✅ 编译失败 |

### 010: 行分隔符终止运行时行为
| 语言 | 代码 | 结果 |
|------|------|------|
| ArkTS | `let x: int = 1` | ✅ |
| Java | `int x = 1;` | ✅ |
| Swift | `let x = 1` | ✅ |

### 011: 分号分隔运行时行为
| 语言 | 代码 | 结果 |
|------|------|------|
| ArkTS | `let x: int = 1; let y: int = 2` | ✅ |
| Java | `int x = 1; int y = 2;` | ✅ |
| Swift | `let x = 1; let y = 2` | ✅ |

---

## 关键结论

1. **语法一致性**：分号语法在 ArkTS、Java、Swift 三种语言中基本一致
2. **行为一致性**：分号分隔和歧义避免的行为在三种语言中一致
3. **跨语言差异**：Java 必须使用分号终止语句，而 ArkTS 和 Swift 支持行分隔符终止
4. **跨语言差异**：分号是三种语言中差异较小的特性之一
