## 四、语言设计差异详情

### 差异 A：行分隔符终止

**现象：**
- ArkTS: 支持行分隔符终止语句（可省略分号）
- Java: 必须使用分号终止语句
- Swift: 支持行分隔符终止语句（可省略分号）

**影响：**
- 语法层面差异
- 运行时语义一致

**规范依据：**
- ArkTS spec §2.11: 声明和语句通常由行分隔符终止
- Java JLS §3.9: 语句必须由分号终止
- Swift Language Reference: 语句可由换行符终止

**分类：** 符合 ArkTS spec 的语言设计差异，非缺陷

---

### 差异 B：歧义避免语法

**现象：**
- ArkTS: `i; -i++` 使用分号避免歧义
- Java: `i; -i++;` 分号终止语句
- Swift: `i; -i += 1` 分号分隔语句

**影响：**
- 语法层面差异
- 功能等价

**规范依据：**
- ArkTS spec §2.11: 分号用于避免歧义
- Java JLS §3.9: 分号终止语句
- Swift Language Reference: 分号分隔同一行的多个语句

**分类：** 符合 ArkTS spec 的语言设计差异，非缺陷
