## 三、语言设计差异详情

### 差异 A：单引号支持

**现象：**
- ArkTS: `'Hello'` (合法)
- Java: `'A'` (仅用于字符)
- Swift: `'Hello'` (与双引号等价)

**影响：**
- 语法层面差异
- 运行时语义一致

**规范依据：**
- ArkTS spec §2.9.6: 单引号和双引号均可用于字符串
- Java JLS §3.10.5: 仅双引号用于字符串
- Swift Language Reference: 单引号和双引号等价

**分类：** 符合 ArkTS spec 的语言设计差异，非缺陷

---

### 差异 B：十六进制转义序列

**现象：**
- ArkTS: `\x41` (合法)
- Java: `\x41` (合法)
- Swift: 不支持

**影响：**
- 语法层面差异
- 功能可通过 `\u{41}` 替代

**规范依据：**
- ArkTS spec §2.9.6: 支持 `\xHH` 转义
- Java JLS §3.10.6: 支持 `\xHH` 转义
- Swift Language Reference: 不支持 `\xHH`

**分类：** 符合 ArkTS spec 的语言设计差异，非缺陷

---

### 差异 C：Unicode 转义语法

**现象：**
- ArkTS: `\u0041` 和 `\u{41}` 均支持
- Java: 仅 `\u0041`
- Swift: 仅 `\u{41}`

**影响：**
- 语法层面差异
- 功能等价

**规范依据：**
- ArkTS spec §2.9.6: 支持两种 Unicode 转义语法
- Java JLS §3.10.6: 仅支持固定 4 位语法
- Swift Language Reference: 仅支持有界语法

**分类：** 符合 ArkTS spec 的语言设计差异，非缺陷

---

### 差异 D：字符串长度方法

**现象：**
- ArkTS: `.length`
- Java: `.length()`
- Swift: `.count`

**影响：**
- API 命名差异
- 功能完全一致

**规范依据：**
- ArkTS spec §2.9.6: 使用 `.length` 属性
- Java JLS: 使用 `.length()` 方法
- Swift Language Reference: 使用 `.count` 属性

**分类：** 符合 ArkTS spec 的语言设计差异，非缺陷

---

### 差异 E：字符串比较语法

**现象：**
- ArkTS: `===` / `!==`
- Java: `.equals()` / `!=`
- Swift: `==` / `!=`

**影响：**
- 语法层面差异
- 语义一致

**规范依据：**
- ArkTS spec §2.8.1: 严格相等运算符用于字符串比较
- Java JLS §15.21.3: 使用 `.equals()` 方法
- Swift Language Reference: 使用 `==` 运算符

**分类：** 符合 ArkTS spec 的语言设计差异，非缺陷

---

### 差异 F：字符串插值语法

**现象：**
- ArkTS: `${expr}`
- Java: `+` 拼接
- Swift: `\(expr)`

**影响：**
- 语法层面差异
- 功能等价

**规范依据：**
- ArkTS spec §2.9.6: 支持 `${expr}` 插值
- Java JLS: 不支持原生插值
- Swift Language Reference: 支持 `\(expr)` 插值

**分类：** 符合 ArkTS spec 的语言设计差异，非缺陷
