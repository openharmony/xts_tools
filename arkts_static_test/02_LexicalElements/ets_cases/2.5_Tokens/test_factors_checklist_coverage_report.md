## 七、Token 测试的跨语言映射说明 ⭐

由于 Tokens 测试属于**词法级别**，Java 和 Swift 的部分语法特性与 ArkTS 不同，因此采用概念映射策略：

### 测试方式说明

| 测试场景 | ArkTS 实现 | Java/Swift 实现策略 | 原因 |
|---------|-----------|-------------------|------|
| `===` 运算 | 包含在最长匹配测试 | 概念映射为 `==` 语义 | Java 没有 `===` |
| Unicode Escape 标识符 | 直接支持 `\u0041var` | 概念映射为 `letA + var` | Java/Swift 语法差异 |
| 常量关键字 `let` | 声明语法 | 概念映射为 `static final` | Java 命名规范差异 |

### 验证核心目标

尽管采用概念映射，Java/Swift 的测试核心目标是验证：

1. ✅ **词法规则一致性**：四种 Token 类型的基本语法规则在三种语言中一致
2. ✅ **运算符语义**：运算符的运行时语义在三种语言中一致
3. ✅ **最长匹配逻辑**：运行的 AST 构建逻辑一致
