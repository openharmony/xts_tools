# 2.9.2 Integer Literals - 测试设计思维导图（v1.1 - 补充cross_lang_verify目录）

> ⚠️ 重要说明：本测试设计参考 ArkTS Static Language Specification §2.9.2，并结合 TESTING_PROCESS_GUIDE.md 要求，标准化为所有章节的统一格式。所有测试用例均已通过实际 ark VM 运行验证。

## 概述
Integer literals 表示没有小数点和指数部分的数字。可以使用 16（十六进制）、10（十进制）、8（八进制）和 2（二进制）进制表示。

**Spec 文法：**
```
IntegerLiteral:
  DecimalIntegerLiteral
  | HexIntegerLiteral
  | OctalIntegerLiteral
  | BinaryIntegerLiteral
;
```

## 全覆盖测试矩阵

### 维度 1：十进制整数

#### A. 基本十进制
- **A1** 零：`0`（001）
- **A2** 正整数：`1`, `153`（001）
- **A3** 最大 int 值：`2147483647`（008）

#### B. 下划线分隔符
- **B1** 十进制带下划线：`1_153`, `1_000_000`（002）

### 维度 2：十六进制整数

#### C. 基本十六进制
- **C1** 十六进制：`0xBAD3`, `0x7F`（003）
- **C2** 十六进制带下划线：`0xBAD_3`（003）
- **C3** 最大 int 十六进制：`0x7FFF_FFFF`（008）

### 维度 3：八进制整数

#### D. 基本八进制
- **D1** 八进制：`0o777`, `0o377`（004）
- **D2** 大写 O：`0O77`（004）

### 维度 4：二进制整数

#### E. 基本二进制
- **E1** 二进制：`0b101`, `0b11111111`（005）
- **E2** 大写 B：`0B1010`（005）

### 维度 5：类型推断

#### F. int 类型推断
- **F1** 值在 0..max(int) 范围内推断为 int（006）

#### G. long 类型推断
- **G1** 值超出 int 范围推断为 long（007）

### 维度 6：边界值

#### H. int 边界
- **H1** max(int) = 2147483647（008）
- **H2** min(int) = -2147483648（009）

#### I. long 边界
- **I1** max(long) = 9223372036854775807（010）
- **I2** min(long) = -max(long) - 1（011）

### 维度 7：compile-fail

#### J. 超出范围
- **J1** max(long) + 1（012）
- **J2** 十六进制超出 long 范围（013）
- **J3** int 类型赋值超出范围（014）
- **J4** int 类型赋值超出负数范围（015）

### 维度 8：runtime

#### K. 运行时验证
- **K1** 各进制值相等（016）
- **K2** 下划线不影响值（017）
- **K3** 类型推断正确性（018）
- **K4** 整数运算正确性（019）
