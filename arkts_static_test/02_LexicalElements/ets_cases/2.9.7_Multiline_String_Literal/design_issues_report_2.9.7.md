## 三、语言设计差异详情

### 差异 A：多行字符串语法

**现象：**
- ArkTS: `` `Line 1\nLine 2` ``
- Java: `"Line 1\nLine 2"` 或 text blocks
- Swift: `"""Line 1\nLine 2"""`

**影响：**
- 语法层面差异
- 运行时语义一致

**规范依据：**
- ArkTS spec §2.9.7: 使用反引号定义多行字符串
- Java JLS §3.10.5: 使用双引号或 text blocks
- Swift Language Reference: 使用三引号

**分类：** 符合 ArkTS spec 的语言设计差异，非缺陷

---

### 差异 B：反引号处理

**现象：**
- ArkTS: 反引号是特殊字符，需要转义 `` \` ``
- Java: 反引号不是特殊字符，直接使用
- Swift: 反引号不是特殊字符，直接使用

**影响：**
- 语法层面差异
- 功能等价

**规范依据：**
- ArkTS spec §2.9.7: 反引号用于定义多行字符串
- Java JLS: 反引号无特殊含义
- Swift Language Reference: 反引号无特殊含义

**分类：** 符合 ArkTS spec 的语言设计差异，非缺陷

---

### 差异 C：行续接语法

**现象：**
- ArkTS: `\` + 换行
- Java: 不支持
- Swift: `\` + 换行

**影响：**
- 语法层面差异
- ArkTS/Swift 支持，Java 不支持

**规范依据：**
- ArkTS spec §2.9.7: 支持行续接
- Java JLS: 不支持行续接
- Swift Language Reference: 支持行续接

**分类：** 符合 ArkTS spec 的语言设计差异，非缺陷
