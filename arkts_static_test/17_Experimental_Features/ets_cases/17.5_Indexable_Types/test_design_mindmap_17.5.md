# 17.5 Indexable Types - 测试设计思维导图

## 概述
ArkTS 17.5 节定义可索引类型（Indexable Types），允许类/接口通过声明特殊方法 `$_get` 和 `$_set` 来启用索引语法。`x[1]` 编译为 `x.$_get(1)`，`x[1] = v` 编译为 `x.$_set(1, v)`。

## 子规则枚举

### 1. 基本 Indexable 定义
- **规则**: 类/接口声明 `$_get(index: Type1): Type2` 和/或 `$_set(index: Type1, value: Type2)`
- **正向编译**: 类声明 $_get 和 $_set，使用 `[]` 语法读写
- **反向编译**: 无对应方法的索引访问
- **运行时**: 实际调用 $_get/$_set，验证返回值正确

### 2. 仅 $_get（只读索引）
- **规则**: 只声明 $_get -> 只支持读访问 `v = x[i]`
- **正向编译**: 声明 $_get 无 $_set，验证读操作可用
- **反向编译**: 对只读索引做写操作 `x[i] = v`
- **运行时**: 读索引返回正确值

### 3. 仅 $_set（只写索引）
- **规则**: 只声明 $_set -> 只支持写访问 `x[i] = v`
- **正向编译**: 声明 $_set 无 $_get，验证写操作可用
- **反向编译**: 对只写索引做读操作 `v = x[i]`
- **运行时**: 写索引正确存储值

### 4. 索引参数类型
- **规则**: 索引参数可以是 string 或 number 类型
- **正向编译**: number 索引、string 索引
- **反向编译**: 非 string/number 类型的索引参数（如 boolean）
- **运行时**: 验证 number 索引和 string 索引行为

### 5. 接口中的抽象定义
- **规则**: $_get/$_set 可以在接口中声明为抽象方法，由实现类提供具体实现
- **正向编译**: 接口声明 $_get，类实现接口并提供方法体
- **反向编译**: 类实现接口但未提供 $_get/$_set 实现
- **运行时**: 通过接口类型变量调用索引

### 6. 继承与重写
- **规则**: $_get/$_set 可以在子类中重写
- **正向编译**: 父类定义 $_get，子类 override 提供不同实现
- **反向编译**: 重写签名不匹配
- **运行时**: 通过父类引用调用子类重写的索引

### 7. 泛型类中使用
- **规则**: $_get/$_set 可以在泛型类中声明和使用
- **正向编译**: 泛型类 `class Box<T>` 中使用 $_get/$_set
- **反向编译**: 类型参数冲突
- **运行时**: 泛型索引读写验证

### 8. async $_get/$_set（应编译失败）
- **规则**: $_get/$_set 标记为 async 应产生编译时错误
- **正向编译**: N/A（不应编译通过）
- **反向编译**: async $_get、async $_set
- **运行时**: N/A

### 9. 重载（Overloading）
- **规则**: $_get 可以用重载签名声明
- **正向编译**: 声明多个 $_get 重载（如 number 和 string 索引）
- **反向编译**: 重载签名冲突
- **运行时**: 不同索引类型调用不同重载

## 分类说明
- **compile-pass** (.ets 文件必须编译成功)
- **compile-fail** (.ets 文件必须产生编译时错误)
- **runtime** (.ets 文件编译并运行，含 main() + assert 断言)

## 文件命名规范
- 前缀: `EXP2_`（第 17 章 Experimental Features）
- 格式: `EXP2_17_05_YYY_{PASS|FAIL|RUNTIME}_{DESCRIPTION}.ets`
- 编号: PASS 001~, FAIL 接续, RUNTIME 接续
- 示例: `EXP2_17_05_001_PASS_BASIC_GET_SET.ets`

## 边界值/异常场景
- $_get 返回 void
- $_set 参数为 any 类型
- 索引操作嵌套 `x[x[0]] = x[1]`
- 空类（无 $_get/$_set）的索引访问
- 多层继承链中的重写
