# 17.12 Default Interface Method Declarations - 测试设计思维导图

## 概述
默认接口方法声明允许在接口中为方法提供默认实现（包含方法体）。实现该接口的类如果不重写该方法，则继承默认实现。默认方法可以声明为 `private`，仅可在接口内部的其余默认方法中调用。

根据 ArkTS Static Spec §17.12：
- 语法：`[private] identifier signature block` 在接口中
- 允许私有默认方法
- block 提供默认实现，实现类若不重写则继承
- 默认方法可在实现类的实例上调用
- 如果类提供了自己的实现，则覆盖默认实现
- 私有默认方法只能在接口的默认方法实现中调用

## 子类型覆盖

### 1. 默认方法基本声明和使用
- **正向编译**: 接口中声明带方法体的默认方法，类实现接口不重写该方法
- **运行时**: 验证默认方法确实被调用，输出预期结果

### 2. 默认方法重写
- **正向编译**: 类重写接口的默认方法，签名匹配
- **运行时**: 验证重写的方法被调用，默认实现被覆盖

### 3. 多个默认方法
- **正向编译**: 接口中声明多个默认方法，类实现该接口
- **运行时**: 验证所有默认方法均可被调用

### 4. 私有默认方法
- **正向编译**: 接口中声明私有默认方法，在另一个默认方法中调用
- **反向编译**: 从实现类外部调用私有默认方法（应报错）
- **运行时**: 验证私有默认方法通过公开默认方法间接调用时的行为

### 5. 默认方法参数和返回值
- **正向编译**: 默认方法带参数和返回值类型
- **运行时**: 验证参数传递和返回值行为

### 6. 默认方法中访问接口属性（this）
- **正向编译**: 默认方法中使用 `this` 访问接口中声明的属性
- **运行时**: 验证 `this` 引用正确指向实现类实例

### 7. 重写签名不兼容
- **反向编译**: 类以不兼容的返回类型重写默认方法（应报错）

### 8. 调用不存在的方法
- **反向编译**: 在实现类实例上调用接口中不存在的方法（应报错）

### 9. 重复默认方法定义
- **反向编译**: 接口中重复声明同名的默认方法（应报错）

### 10. 无效语法
- **反向编译**: 默认方法声明语法不完整（缺少闭合大括号等）

### 11. 复杂运行时逻辑
- **运行时**: 验证默认方法中的条件分支、循环、属性访问等复杂逻辑

## 分类说明
- **compile-pass**：文件必须编译通过（es2panda 退出码 0）
- **compile-fail**：文件必须产生编译错误（es2panda 退出码非 0）
- **runtime**：文件编译通过 + ark VM 实际运行通过 + assert 断言验证

## 文件命名规范
- 前缀：`EXP2_`（17_Experimental_Features 章节前缀）
- 子章节号：`17_12`
- 类别：`PASS` / `FAIL` / `RUNTIME`
- 编号：001-005 (compile-pass), 010-014 (compile-fail), 020-023 (runtime)
- 示例：`EXP2_17_12_001_PASS_DEFAULT_METHOD_BASIC.ets`
