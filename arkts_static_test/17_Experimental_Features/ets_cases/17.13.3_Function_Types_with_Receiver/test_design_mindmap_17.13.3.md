# 17.13.3 Function Types with Receiver - 测试设计思维导图

## 概述
函数类型可以带有接收者，语法为 `(this: ReceiverType, paramList?) => ReturnType`。只有带接收者的函数类型才能用于方法调用语法。可以泛型化。函数类型兼容性忽略参数名称。可赋值给变量、作为参数传递。

## 子类型覆盖

### 1. 基本函数类型声明
- **正向编译**: 声明带接收者的函数类型
- **正向编译**: 声明变量使用带接收者的函数类型并赋值 lambda
- **正向编译**: 通过函数类型变量调用（方法语法和普通语法）
- **正向编译**: 函数类型作为参数传递

### 2. 泛型函数类型
- **正向编译**: 泛型函数类型声明
- **正向编译**: 泛型接收者函数类型变量赋值和调用

### 3. 类型兼容性
- **反向编译**: 非接收者函数赋值给接收者函数类型变量
- **反向编译**: 接收者类型不匹配的赋值
- **反向编译**: 参数数量不匹配

### 4. 调用约束
- **反向编译**: 不带接收者的函数类型不能使用方法调用语法
- **反向编译**: 带接收者的函数类型方法调用需要正确的接收者类型
