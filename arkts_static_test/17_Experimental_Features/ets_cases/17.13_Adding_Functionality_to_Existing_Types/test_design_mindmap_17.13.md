# 17.13 Adding Functionality to Existing Types - 测试设计思维导图

## 概述
本章节概述通过接收者函数（functions with receiver）为已有类型添加功能的概念。接收者函数是一种顶层函数，其第一个参数名为 `this`，从而可以作为该类型的"扩展方法"调用。

## 子类型覆盖

### 1. 基本接收者函数定义
- **正向编译**: 定义一个接收者为类类型的函数，通过方法调用语法和普通函数调用语法调用
- **反向编译**: 接收者类型不兼容的情况
- **运行时**: 验证接收者函数的实际执行效果

### 2. 接收者函数为接口类型添加功能
- **正向编译**: 接收者函数作用于接口类型，实现该接口的类实例均可使用
- **反向编译**: N/A（此子章节为概述性质）
- **运行时**: 验证接口接收者函数的运行时行为

### 3. 多个接收者函数作用于同一类型
- **正向编译**: 为同一类型定义多个接收者函数
- **反向编译**: N/A
- **运行时**: 验证多个接收者函数的独立工作

## 测试点汇总

| 分类 | 测试点 | 期望结果 |
|------|--------|---------|
| compile-pass | 基本类接收者函数定义和方法调用 | 编译通过 |
| compile-pass | 接口类型接收者函数 | 编译通过 |
| compile-pass | 同一类型的多个接收者函数 | 编译通过 |
| compile-pass | 接收者函数的普通函数调用语法 | 编译通过 |
| compile-fail | 接收者类型与参数类型不兼容 | 编译错误 |
| compile-fail | 对不兼容类型使用接收者方法调用 | 编译错误 |
| runtime | 验证接收者函数运行时正确执行 | 断言通过 |
| runtime | 验证普通函数调用语法运行时执行 | 断言通过 |
