# 17.11.3 Named Constructors - 测试设计思维导图

## 概述
命名构造函数是 ArkTS 实验特性，允许为构造函数赋予名称。语法：`constructor Name(params) { ... }`。命名构造函数通过 `overload constructor { Name1, Name2, ... }` 声明加入重载集合。调用时通过 `new ClassName(args)` 进行重载解析（编译器根据参数类型/数量匹配正确的命名构造函数）。

## 子规则完整枚举

### 1. 命名构造函数基本声明
- **正向编译**: 声明单个命名构造函数，声明多个命名构造函数
- **反向编译**: 无对应场景
- **运行时**: 调用命名构造函数创建对象并验证状态

### 2. 命名构造函数与匿名构造函数共存
- **正向编译**: 匿名构造函数 + 命名构造函数共存，overload constructor 块包含命名构造函数
- **反向编译**: 无对应场景
- **运行时**: 验证匿名和命名构造函数分别正确解析

### 3. 命名构造函数重载解析
- **正向编译**: 多个命名构造函数通过参数类型/数量区分
- **反向编译**: 两个命名构造函数同名且参数完全相同
- **运行时**: 验证不同参数匹配不同构造函数

### 4. 全命名构造函数（无匿名构造函数）
- **正向编译**: 类只有命名构造函数，无匿名构造函数
- **反向编译**: 无匿名构造函数时调用 new Foo() 无匹配参数
- **运行时**: 验证只能通过匹配参数的调用方式

### 5. 构造函数名作为引用
- **正向编译**: 无对应场景
- **反向编译**: 将构造函数名作为属性引用（如 Foo.Bar）
- **运行时**: 无对应场景

### 6. overload constructor 块
- **正向编译**: overload constructor 块声明
- **反向编译**: 同一个类中两个 overload constructor 块
- **运行时**: 验证 overload 解析顺序

### 7. 命名构造函数与继承
- **正向编译**: 父类有命名构造函数，子类有自己的构造函数
- **反向编译**: 无对应场景
- **运行时**: 验证子类对象创建
