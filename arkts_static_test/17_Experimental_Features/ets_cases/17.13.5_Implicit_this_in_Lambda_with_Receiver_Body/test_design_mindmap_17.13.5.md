# 17.13.5 Implicit this in Lambda with Receiver Body - 测试设计思维导图

## 概述
规范描述在接收者lambda体内可以省略 `this.` 前缀直接访问接收者成员。实际 es2panda 实测发现：当前编译器**不支持**隐式 `this`，必须使用显式 `this.field` 访问。因此本章节用例调整为验证显式 `this` 的正确行为。

## 子类型覆盖

### 1. 显式 this 访问（实际行为）
- **正向编译**: 显式 this.field 访问接收者成员
- **正向编译**: 显式 this 调用接收者方法
- **正向编译**: 接收者函数中使用显式 this
- **运行时**: 验证显式 this 正确解析

### 2. 非法访问（反向编译）
- **反向编译**: 访问不存在的成员（隐式访问不存在的名称）
- **反向编译**: 访问私有成员
- **反向编译**: 外部作用域变量与接收者成员歧义（隐式访问时）

## 实测发现
- ⚠️ es2panda 不支持隐式 this（省略 `this.` 前缀）
- 所有成员访问必须使用显式 `this.` 前缀
- 这与规范 §17.13.5 描述不同
