## 三、符合 ArkTS spec 的语言设计差异与观察

### 观察 A：内置数组明确排除在命名类型外 ⭐ DESIGN

**spec §3.4 原文：**
> Named types are classes, interfaces, enumerations, aliases, type parameters, and predefined types (see Predefined Types), **except built-in arrays**.

**为何独立设计？**
- 内置数组 `int[]` 和 `Array<int>` 在语法上等价，但实现差异
- ArkTS 为支持类型擦除 + 类型推断的特定行为可能需要这种特殊处理

**对比：**
| 语言 | 数组归类 |
|------|---------|
| ArkTS | ❌ 不是命名类型 |
| Java | ✅ 是命名类型 |
| Swift | ✅ 是命名类型 |

**评价：** 此设计在 spec 中无详细解释，但运行时和编译期行为正常。建议 spec 添加注释说明设计动机。

### 观察 B：ArkTS 比 Java 更严格地禁止 raw type

**用例：** TYP_03_04_016

```typescript
class GenericC<T> {}
let c: GenericC = new GenericC(1)   // ❌ ArkTS 编译失败
```

```java
class GenericC<T> {}
GenericC c = new GenericC();   // ⚠️ Java 仅警告
```

**评价：** ArkTS 设计更安全，避免了 Java raw type 的运行时类型问题。
