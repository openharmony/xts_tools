# 3.2 User-Defined Types - 测试设计思维导图

## 概述
用户定义类型是开发者在 ArkTS 中通过声明引入的类型，包括类、接口、枚举、函数类型、元组、联合、类型参数和字面量类型。

## 子类型覆盖

### 1. 类类型（Class Types）
- 正向编译: 类声明、new 实例化、成员访问、继承
- 反向编译: 抽象类实例化、缺失方法实现
- 运行时: 继承链方法调用

### 2. 接口类型（Interface Types）
- 正向编译: 接口声明、implements 实现
- 反向编译: 未实现接口方法
- 运行时: 接口多态

### 3. 枚举类型（Enumeration Types）
- 正向编译: enum 声明、成员引用、值访问
- 反向编译: 非法成员名、跨枚举比较
- 运行时: values()/fromValue() 方法

### 4. 函数类型（Function Types）
- 正向编译: type 别名函数签名、变量声明、赋值
- 反向编译: 参数数量/类型不匹配

### 5. 元组类型（Tuple Types）
- 正向编译: [type1, type2] 声明、索引访问
- 反向编译: 元素类型不匹配、越界访问

### 6. 联合类型（Union Types）
- 正向编译: T1 | T2 声明、赋值、类型收窄
- 反向编译: 无效的联合成员赋值

### 7. 类型参数（Type Parameters）
- 正向编译: 泛型类/函数 <T>、类型约束 extends
- 反向编译: 超出约束的类型参数

### 8. 字面量类型（Literal Types）
- 正向编译: "hello" 字面量类型、42 字面量类型
- 反向编译: 不匹配的字面量赋值

## 分类说明
- **compile-pass**: 编译通过的正向用例
- **compile-fail**: 应产生编译时错误的反向用例
- **runtime**: 测试运行时行为的用例
